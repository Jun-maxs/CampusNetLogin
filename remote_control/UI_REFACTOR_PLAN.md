# UI 重构临时计划

## 目标
- 重构服务器 UI，按钮按功能归类，入口统一为“点击机子进入详情操作”
- 保留现有全部功能，不丢命令、不丢状态、不丢更新能力
- 将标题从“校园网远程控制面板”改为“减肥就喝健力宝”
- 按 `ui-ux-pro-max` 风格进行视觉重构
- 健全日志系统，支持分类查看、按设备筛选、查看单机专属日志

## Plan
- [ ] 1. 盘点现有 UI 与命令能力，确认必须保留的功能全集
- [ ] 2. 重构列表页：卡片仅保留核心信息，点击进入单机详情抽屉/面板
- [ ] 3. 重构单机详情页：按分组展示全部操作（网络、限速/DNS、自启/防护、生命周期、更新）
- [ ] 4. 重构日志系统：会话日志 + 历史日志，支持分类、设备筛选、搜索
- [ ] 5. 测试所有关键操作链路，完成后删除本计划文件

## 执行前 Rules
- 必须先梳理当前所有命令与状态字段，避免 UI 重构时遗漏功能
- 不修改 Agent / Server API 协议语义，除非确有必要且同步兼容
- 所有新 UI 必须优先保证可用性，其次再追求视觉效果
- 保持现有更新、DNS、限速、下线、自启、防护、卸载、删除、日志相关能力完整
- 尽量只改 `remote_control/server.py`，避免无关文件扰动

## 执行中 Rules
- 卡片层只显示摘要信息，复杂操作统一收纳到详情抽屉/面板
- 操作必须按语义分组，避免按钮散乱堆叠
- 所有危险操作必须继续保留确认流程
- 任何功能若从主卡片移走，必须在详情区有明确入口
- 日志系统必须可按设备过滤，且默认可查看全局日志
- 保持页面刷新策略稳定，避免频繁重绘和闪烁

## 执行后检验 Rules
- 检查每个旧功能是否仍可从新 UI 到达并触发
- 检查单机详情中是否能完成：下线、解锁、刷新、取消无感、限速、解除限速、设置 DNS、恢复 DNS、自启开关、防护、卸载、删除、推送更新
- 检查日志是否支持：全部日志、单机日志、分类查看、关键字搜索
- 检查标题与视觉样式是否已更新为新方案
- 检查无 JS 报错、无明显布局错乱、无核心功能丢失
- 全部完成后删除本文件 `UI_REFACTOR_PLAN.md`
